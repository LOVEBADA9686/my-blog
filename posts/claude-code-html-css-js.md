클로드 코드와 함께 이 블로그를 만들면서, 세 가지 언어가 얼마나 명확하게 역할을 나눠 맡고 있는지 새삼 느꼈다. 오늘 배운 걸 간단히 정리해본다.

## HTML: 뼈대와 의미

`index.html`과 `post.html`은 딱 두 페이지뿐이지만, 이 둘이 사이트의 뼈대를 이룬다. HTML은 "여기에 글 목록이 들어간다", "여기에 본문이 들어간다"처럼 구조와 의미를 정의하는 역할만 한다. 실제 내용은 비어있고, JS가 나중에 채워 넣는다.

```html
<main id="post-list"></main>
```

## CSS: 테마와 반응형

`css/style.css`는 CSS 변수로 라이트/다크 테마를 정의한다. `--bg`, `--text`, `--accent` 같은 변수를 `prefers-color-scheme` 미디어 쿼리로 기본값을 잡고, 토글 버튼으로 덮어쓸 수 있게 만들었다.

```css
:root {
  --bg: #ffffff;
  --text: #1a1a1a;
  --accent: #ff6b6b;
}

@media (prefers-color-scheme: dark) {
  :root {
    --bg: #1a1a1a;
    --text: #f0f0f0;
  }
}
```

모바일 우선으로 작성하고, 화면이 넓어질수록 여백을 늘리는 미디어 쿼리를 추가하는 순서로 작업하니 반응형을 훨씬 수월하게 다룰 수 있었다.

## JavaScript: 동작과 데이터

`js/render.js`는 마크다운 파일을 `fetch()`로 읽어와 front matter를 파싱하고 HTML로 렌더링한다. `js/app.js`는 `manifest.json`을 불러와 목록을 그리고 태그 필터와 검색을 처리한다. `js/theme.js`는 다크모드 상태를 `localStorage`에 저장해 새로고침해도 유지되게 한다.

빌드 도구 없이 브라우저에서 실시간으로 마크다운을 파싱한다는 제약이 있었지만, 오히려 각 스크립트의 책임이 명확하게 나뉘어서 코드를 이해하기 쉬웠다.

## 정리

- **HTML**: 구조와 의미 (뼈대)
- **CSS**: 표현과 테마 (스타일)
- **JavaScript**: 데이터와 동작 (로직)

프레임워크 없이 세 언어의 기본 역할만 충실히 지켜도 꽤 쓸만한 사이트를 만들 수 있다는 걸 클로드 코드와 함께 작업하며 다시 확인했다.
